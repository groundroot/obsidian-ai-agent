## 이 포크 버전에서 달라진 점

이 저장소는 기존 OSBA의 핵심 기능을 유지하면서, **클라우드 AI뿐 아니라 로컬 AI도 실사용 가능하도록 확장한 Ollama 버전**입니다.

### 포크 버전 핵심 목표

- Ollama를 통한 **로컬 생성 모델 / 로컬 임베딩 모델** 지원
- BRAT 기준으로 설치 가능한 배포 흐름 정리
- Mac 사용자용 **MLX 안내 및 Gemma 추천 흐름** 추가
- 인덱싱과 연결분석 상태를 `osba` frontmatter로 추적
- 이미 최신 상태인 노트는 인덱싱/분석을 다시 하지 않도록 최적화

### 이 포크에서 새롭게 추가된 로컬 모델 기능

이제 이 플러그인은 **Ollama 서버를 통해 로컬 LLM을 사용할 수 있습니다.**

즉, 다음과 같은 흐름이 가능합니다:

- 로컬 생성 모델로 Quick Draft
- 로컬 생성 모델로 연결분석
- 로컬 임베딩 모델로 유사 노트 검색
- 로컬 전용 구성으로 현재 노트 인덱싱 / 전체 인덱싱

### 지원 방식

- **Ollama OpenAI 호환 API** 지원
  - `/v1/chat/completions`
  - `/v1/embeddings`
- **Ollama native API** fallback 지원
  - `/api/chat`
  - `/api/embed`
  - `/api/embeddings`
  - `/api/tags`

즉, Ollama가 실행 중이면 이 플러그인은 **Gemma 계열 모델을 포함해 Ollama에서 제공하는 모든 로컬 모델을 생성 모델 또는 임베딩 모델로 활용**할 수 있습니다.

### 추천 로컬 모델 예시

- 생성 모델
  - `gemma4:e4b`
  - `mlx-community/gemma-2-2b-it`
  - `llama3.1:8b`
  - `qwen2.5:7b`
- 임베딩 모델
  - `nomic-embed-text`

**참고**
- `gemma4:e4b`는 품질 우선 추천
- `mlx-community/gemma-2-2b-it`는 속도 우선 추천
- 임베딩은 `nomic-embed-text` 계열을 권장합니다

## 이 플러그인이 뭔가요? (쉬운 설명)

### 비유 1: 도서관의 슈퍼 사서

여러분의 Obsidian 볼트를 **거대한 도서관**이라고 상상해보세요.

| 비유 | 실제 의미 |
|------|-----------|
| 도서관 | Obsidian 볼트 |
| 책들 | 노트 파일들 |
| 슈퍼 사서 | OSBA 플러그인 |

**일반 도서관에서는...**
- 책 제목이나 저자를 정확히 알아야 찾을 수 있어요
- "비슷한 내용의 책 찾아주세요"가 어려워요
- 새 책이 어떤 책과 관련있는지 사서도 잘 몰라요

**OSBA 사서가 있으면...**
- "AI에 대해 적은 노트가 어디있지?" → 바로 찾아줌!
- "이 노트랑 비슷한 거 있어?" → 의미가 비슷한 노트 추천!
- "내가 뭘 모르는지 알려줘" → 지식 갭 분석!

### 비유 2: 노트에 붙이는 "내용 바코드"

마트에서 물건에 바코드를 찍으면 가격, 정보가 나오죠?

**OSBA는 모든 노트에 "내용 바코드"를 붙입니다.**

```text
일반 검색: 글자가 똑같아야 찾음
           "생산성" 검색 → "생산성"이라는 글자가 있는 노트만

OSBA 검색: 의미가 비슷하면 찾음
           "생산성" 검색 → "효율성", "시간관리", "업무최적화" 노트도!
