## 이 포크 버전에서 달라진 점

이 저장소는 기존 OSBA의 핵심 기능을 유지하면서, **클라우드 AI뿐 아니라 로컬 AI도 실사용 가능하도록 확장한 포크 버전**입니다.

### 포크 버전 핵심 목표

- Ollama를 통한 **로컬 생성 모델 / 로컬 임베딩 모델** 지원
- BRAT 기준으로 설치 가능한 배포 흐름 정리
- Mac 사용자용 **MLX 안내 및 Gemma 추천 흐름** 추가
- 인덱싱과 연결분석 상태를 `osba` frontmatter로 추적
- 이미 최신 상태인 노트는 인덱싱/분석을 다시 하지 않도록 최적화

### 이 포크에서 새롭게 추가된 로컬 모델 기능

이제 이 플러그인은 **Ollama 서버를 통해 로컬 LLM을 사용할 수 있습니다.**

즉, 다음과 같은 흐름이 가능합니다:

- 로컬 생성 모델로 Quick Draft
- 로컬 생성 모델로 연결분석
- 로컬 임베딩 모델로 유사 노트 검색
- 로컬 전용 구성으로 현재 노트 인덱싱 / 전체 인덱싱

### 지원 방식

- **Ollama OpenAI 호환 API** 지원
  - `/v1/chat/completions`
  - `/v1/embeddings`
- **Ollama native API** fallback 지원
  - `/api/chat`
  - `/api/embed`
  - `/api/embeddings`
  - `/api/tags`

즉, Ollama가 실행 중이면 이 플러그인은 **Gemma 계열 모델을 포함해 Ollama에서 제공하는 모든 로컬 모델을 생성 모델 또는 임베딩 모델로 활용**할 수 있습니다.

### 추천 로컬 모델 예시

- 생성 모델
  - `gemma4:e4b`
  - `mlx-community/gemma-2-2b-it`
  - `llama3.1:8b`
  - `qwen2.5:7b`
- 임베딩 모델
  - `nomic-embed-text`

**참고**
- `gemma4:e4b`는 품질 우선 추천
- `mlx-community/gemma-2-2b-it`는 속도 우선 추천
- 임베딩은 `nomic-embed-text` 계열을 권장합니다

## 이 플러그인이 뭔가요? (쉬운 설명)

### 비유 1: 도서관의 슈퍼 사서

여러분의 Obsidian 볼트를 **거대한 도서관**이라고 상상해보세요.

| 비유 | 실제 의미 |
|------|-----------|
| 도서관 | Obsidian 볼트 |
| 책들 | 노트 파일들 |
| 슈퍼 사서 | OSBA 플러그인 |

**일반 도서관에서는...**
- 책 제목이나 저자를 정확히 알아야 찾을 수 있어요
- "비슷한 내용의 책 찾아주세요"가 어려워요
- 새 책이 어떤 책과 관련있는지 사서도 잘 몰라요

**OSBA 사서가 있으면...**
- "AI에 대해 적은 노트가 어디있지?" → 바로 찾아줌!
- "이 노트랑 비슷한 거 있어?" → 의미가 비슷한 노트 추천!
- "내가 뭘 모르는지 알려줘" → 지식 갭 분석!

---

### 비유 2: 노트에 붙이는 "내용 바코드"

마트에서 물건에 바코드를 찍으면 가격, 정보가 나오죠?

**OSBA는 모든 노트에 "내용 바코드"를 붙입니다.**

```
일반 검색: 글자가 똑같아야 찾음
           "생산성" 검색 → "생산성"이라는 글자가 있는 노트만

OSBA 검색: 의미가 비슷하면 찾음
           "생산성" 검색 → "효율성", "시간관리", "업무최적화" 노트도!
```

이 바코드를 **"임베딩(Embedding)"**이라고 불러요.
AI가 노트의 의미를 숫자로 변환해서 저장하는 거예요.

---

### 비유 3: 관계 전문가 친구

500개의 노트가 있다고 상상해보세요.
사람이라면 다 읽고 관계 파악하는 데 며칠이 걸리겠죠?

**OSBA는 AI를 사용해서 빠르게 분석합니다:**

```
📝 "프로젝트 관리" 노트를 분석하면...

├── ✨ "GTD 방법론" 노트를 확장합니다
│       (이 노트가 GTD를 더 자세히 설명하고 있어요)
│
├── 🔄 "애자일 개발" 노트와 관련있습니다
│       (비슷한 주제를 다루고 있어요)
│
└── 💡 "일정 관리" 노트를 뒷받침합니다
        (예시나 근거를 제공해요)
```

---

### 비유 4: 학습 코치

OSBA는 당신의 노트를 분석해서 **부족한 부분**도 알려줍니다:

```
🔍 발견된 지식 갭 (Gap):

1. "머신러닝 기초" 노트를 보니까...
   → "딥러닝"에 대해 더 공부하면 좋겠어요! (우선순위: 높음)

2. "투자 포트폴리오" 노트를 보니까...
   → "리스크 관리" 지식이 부족해 보여요 (우선순위: 중간)
```

마치 선생님이 "이 부분 더 공부해봐"라고 알려주는 것처럼요!

## 핵심 기능 4가지

### 1. 의미 기반 검색 (Similar Notes)

| 기존 검색 | OSBA 검색 |
|-----------|-----------|
| 글자가 일치해야 함 | 의미가 비슷하면 OK |
| "인공지능" → "인공지능" 노트만 | "인공지능" → "AI", "머신러닝", "딥러닝" 노트도! |

### 2. 자동 연결 발견 (Analyze Note)

- 관련 노트를 자동으로 찾아줌
- 관계 유형도 알려줌 (확장/지원/반박/예시/관련)
- 확신도 점수 제공 (예: 85% 확신)

### 3. 지식 갭 분석 (Knowledge Gaps)

- 노트에서 부족한 부분 발견
- 더 공부할 주제 추천
- 우선순위로 정리 (높음/중간/낮음)

### 4. Quick Draft (빠른 초안)

```
사용자: "AI 윤리에 대해 노트 작성하고 싶어"

OSBA: [기존 노트들을 참고해서...]
      - "AI 기술 트렌드" 노트 참고
      - "기술 윤리학" 노트 참고

      [초안 생성 완료!]
```

기존 노트의 맥락을 활용해서 새 노트 초안을 만들어줍니다.

## 설치 방법

### 방법 1: BRAT 사용 (추천)

**BRAT = 베타 플러그인 설치 도구**

```
1단계: BRAT 설치
├── Obsidian 열기
├── 설정 → 커뮤니티 플러그인 → 찾아보기
├── "BRAT" 검색 → 설치 → 활성화
└── 완료!

2단계: OSBA 추가
├── 설정 → BRAT
├── "Add Beta plugin" 클릭
├── 입력: 저장소 URL
│   예: https://github.com/groundroot/obsidian-ai-agent
└── 완료!

3단계: 활성화
├── 설정 → 커뮤니티 플러그인
├── "obsidian-ai-agent" 찾기
└── 토글 켜기!
```

**중요**
- BRAT은 저장소 코드가 아니라 **GitHub Release 자산**을 설치합니다
- 따라서 새 버전을 배포할 때는 반드시 `main.js`, `manifest.json`, `styles.css`를 릴리즈에 업로드해야 합니다

### 방법 2: 수동 설치

1. GitHub 릴리즈에서 `main.js`, `manifest.json`, `styles.css` 다운로드
2. 볼트의 `.obsidian/plugins/obsidian-ai-agent/` 폴더 만들기
3. 다운로드한 파일들 복사
4. Obsidian 재시작

## 초기 설정 (처음 사용자용)

### Step 1: API 키 발급

**API 키 = AI 서비스 이용권**

마치 넷플릭스 구독권처럼, AI 서비스를 이용하려면 "이용권"이 필요해요.

| 서비스 | 용도 | 필수? | 발급처 |
|--------|------|-------|--------|
| OpenAI | 클라우드 임베딩 / OpenAI 모델 사용 | 선택 | [platform.openai.com](https://platform.openai.com) |
| Gemini | 빠른 초안 작성 | 선택 | [aistudio.google.com](https://aistudio.google.com) |
| Claude | 깊은 분석 | 선택 | [console.anthropic.com](https://console.anthropic.com) |
| xAI Grok | 초안/분석 (128K 컨텍스트) | 선택 | [console.x.ai](https://console.x.ai) |
| Ollama | 로컬 생성 / 로컬 임베딩 | 선택 | [ollama.com](https://ollama.com) |

**최소 구성**
- 클라우드만 사용할 때: OpenAI API 키만 있으면 기본 기능 사용 가능
- 로컬만 사용할 때: Ollama 설치 + 생성 모델 + 임베딩 모델 설정만으로 사용 가능

### Step 2: API 키 입력

```
1. Obsidian → 설정 → obsidian-ai-agent
2. 🔑 API 키 설정 섹션에서 키 입력
3. 각 키 옆 "테스트" 버튼 클릭
4. "✅ 연결 성공!" 확인
```

### Step 2-1: Ollama 로컬 모델 설정

```
1. Ollama 설치
2. 터미널에서 모델 다운로드
   예: ollama pull gemma4:e4b
   예: ollama pull mlx-community/gemma-2-2b-it
   예: ollama pull nomic-embed-text
3. Obsidian → 설정 → obsidian-ai-agent
4. 🦙 Ollama 사용 켜기
5. Base URL 확인 (기본값: http://localhost:11434)
6. Generation Model / Embedding Model 선택
7. Gemma 프리셋 또는 수동 선택
8. "연결 테스트"로 확인
```

**추천 조합**
- 생성: `gemma4:e4b`, `mlx-community/gemma-2-2b-it`, `llama3.1:8b`, `qwen2.5:7b`
- 임베딩: `nomic-embed-text`, `nomic-embed-text:latest`

**Mac 사용자 안내**
- 이 포크 버전에는 Mac 사용자를 위한 MLX 안내가 포함되어 있습니다
- 플러그인이 직접 MLX 런타임을 실행하지는 않지만, Ollama + Apple Silicon 환경에서 빠르게 쓸 수 있는 모델 선택 흐름을 제공합니다
- 속도 우선 추천: `mlx-community/gemma-2-2b-it`
- 최신 Gemma 추천: `gemma4:e4b`

### Step 3: 인덱싱 모드 선택 (NEW!)

**인덱싱 = 노트에 바코드 붙이기**

#### 제외 모드 (기본값)
```
"이 폴더들만 빼고 전부 인덱싱해줘"

예시:
├── templates (제외) ❌
├── .obsidian (제외) ❌
├── notes (인덱싱) ✅
├── projects (인덱싱) ✅
└── journal (인덱싱) ✅
```

#### 포함 모드
```
"이 폴더들만 인덱싱해줘"

예시:
├── templates (무시)
├── .obsidian (무시)
├── notes (인덱싱) ✅
├── projects (인덱싱) ✅
└── journal (무시)
```

**언제 어떤 모드?**

| 상황 | 추천 모드 |
|------|-----------|
| 노트 많고, 몇 개만 제외하고 싶을 때 | 제외 모드 |
| 특정 폴더만 AI로 분석하고 싶을 때 | 포함 모드 |
| 개인 노트는 AI 분석에서 빼고 싶을 때 | 제외 모드 + private 폴더 제외 |

## 사용 방법

### 명령어 팔레트 (Cmd+P / Ctrl+P)

"OSBA"로 검색하면 모든 명령어가 나와요:

| 명령어 | 설명 | 언제 쓸까? |
|--------|------|-----------|
| Quick Draft | AI 초안 생성 | 새 노트 작성 시작할 때 |
| Analyze Current Note | 현재 노트 분석 | 관련 노트 찾고 싶을 때 |
| Find Similar Notes | 비슷한 노트 검색 | 연관 자료 찾을 때 |
| Batch Index Vault | 전체 인덱싱 | 처음 설정할 때 |
| Batch Analyze Vault | 전체 연결분석 | 볼트 전체 관계를 다시 분석할 때 |

### 자동 기능 (설정에서 ON/OFF)

- **자동 분석**: 새 노트 만들면 자동으로 관련 노트 찾기
- **자동 임베딩**: 노트 수정하면 자동으로 바코드 업데이트
- **인덱싱 후 자동 연결분석**: 현재 노트 인덱싱/전체 인덱싱 후 연결분석까지 이어서 수행

### 작업 흐름 이해하기

이 포크 버전에서는 인덱싱과 분석을 분리해서 생각하면 이해가 쉽습니다.

#### 1. 인덱싱

- 노트의 의미를 임베딩으로 변환
- 유사 노트 검색과 RAG 기반 초안 생성 준비
- `osba.embeddingHash`, `osba.indexedAt`, `osba.embeddingModel` 기록

#### 2. 연결분석

- 현재 노트와 관련 노트의 관계를 AI가 해석
- 연결 유형, 지식 갭, 인사이트 생성
- `osba.analyzed`, `osba.analyzedAt`, `osba.analysisHash` 기록

#### 3. 자동 스킵 로직

플러그인은 `osba` frontmatter와 캐시 데이터를 함께 사용해 다음을 판단합니다.

- 이미 최신 상태로 인덱싱된 노트인가?
- 이미 최신 상태로 분석된 노트인가?
- 노트 내용이 바뀌어서 stale 상태인가?

내용 해시가 같으면:
- 인덱싱 다시 하지 않음
- 연결분석 다시 하지 않음

내용이 바뀌면:
- `osba.indexStatus`가 `stale`로 바뀜
- 필요 시 다시 인덱싱 / 다시 분석

### 작업 대기열

이 포크 버전에서는 다음 작업들이 작업 대기열에 표시됩니다.

- 연결 분석
- 빠른 초안
- 유사 노트 찾기
- 현재 노트 인덱싱
- 전체 인덱싱
- 전체 연결분석

작업 대기열은 이제 전체 화면을 계속 다시 그리지 않고, **새로운 작업이나 상태 변화가 있을 때만 갱신**되도록 개선되었습니다.

## 비용 이해하기

### API 비용 = 통화료

**비유: 전화 통화**

OSBA를 사용하면 AI 서비스에 "전화"를 겁니다.
전화를 많이 할수록 요금이 나와요.

```
임베딩 (바코드 만들기)
```

## 문제 해결

### "API 키가 유효하지 않습니다"

```
체크리스트:
□ 키가 정확히 복사됐나요?
□ 앞뒤 공백은 없나요?
□ 해당 서비스에 결제 수단 등록했나요?
□ 테스트 버튼 눌러봤나요?
```

### "인덱싱이 안 돼요"

```
체크리스트:
□ 클라우드 모드라면 OpenAI API 키 설정했나요?
□ 로컬 모드라면 Ollama가 실행 중인가요?
□ 로컬 임베딩 모델로 `nomic-embed-text` 계열을 선택했나요?
□ 인덱싱할 폴더가 제외 목록에 있진 않나요?
□ 노트 크기가 최대 허용 크기 이하인가요?
□ 포함 모드인데 폴더 지정 안 했나요?
```

### "비용이 너무 많이 나왔어요"

```
해결책:
1. 예산 한도 설정하기
2. 자동 분석/임베딩 끄기
3. 저렴한 모델로 변경 (Gemini Flash)
4. 불필요한 폴더 제외하기
5. Ollama 로컬 모델 사용하기
```

### "왜 예전 노트는 다시 인덱싱/분석하지 않나요?"

이 포크 버전은 **최신 상태 검증 후 스킵**하는 방식입니다.

확인 기준:
- `osba.indexedAt`
- `osba.embeddingHash`
- `osba.analyzed`
- `osba.analyzedAt`
- `osba.analysisHash`

즉, 노트 내용이 바뀌지 않았다면 불필요한 중복 작업을 하지 않는 것이 정상입니다.
